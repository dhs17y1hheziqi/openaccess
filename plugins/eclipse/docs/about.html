Versant OpenAccess Eclipse Plugin



www.versant.com

  

Versant OpenAccess is a high performance Java Data Objects implementation supporting leading commercial and open source relational databases. This plugin integrates JDO enhancement into the Eclipse build process. The standalone OpenAccess Workbench is used to edit mappings and to setup a project (run workbench.bat or workbench.sh in the distribution).

Using OpenAccess with an Eclipse project

Right click the project that you want to use and select Versant OpenAccess | Add JDO nature on the popup menu. (A 'OpenAccess Libs' classpath container is added to you project's 'Java Build Path'. This classpath container contains the openaccess.jar and jta.jar)

Select Project | Properties | OpenAccess Properties and configure your project for OpenAccess. The fields on the form are as follows:

  1. Project file name. Specifies the OpenAccess (versant.properties) project file.

  2. Auto Enhance. Enhance persistent classes whenever they are recompiled. The Enhancer is added to the 'External Tools Builders' section. You can always enhance manually by selecting Versant OpenAccess | Enhance classes from the project popup menu.

  3. Copy project file to output dir. Copy the project file to the project classpath on enhancement. This is useful if the project file is not in the classpath or if you want to use an Ant style token replacement.

  4. Ant filter token properties. This is used to resolve Ant filter tokens in your OpenAccess project file when it is copied to the classpath. This option is only available if 'Copy project file to output dir' is selected.

  5. Use relative paths. All paths are stored relative the project if this is on.

For more information visit www.versant.com.
